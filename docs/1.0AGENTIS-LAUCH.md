Exactly What You Build in Each Week
Week 1 — Tauri shell
Install Tauri CLI. Run npm create tauri-app. Point it at your existing apps/api as the backend process to spawn and localhost:3737 as the URL to open. Tauri's sidecar feature handles spawning your Node.js API server as a child process when the app launches. The app icon appears in the system tray. Clicking it opens the browser. That's the entire week — it's mostly configuration, not code.
Week 2 — installers and polish
Tauri's tauri build generates .dmg for Mac, .exe / .msi for Windows, .deb / .AppImage for Linux automatically. You configure code signing for Mac — which requires an Apple Developer account at $99/year, non-negotiable for distribution. Set up GitHub Actions to build all three targets on every release tag. This is your entire CI pipeline for desktop distribution. Add the auto-updater — also built into Tauri, points at a GitHub releases endpoint, checks on app startup.
Week 3 — landing page and the three cloud endpoints
Build agentis.dev as a simple static site. Three API endpoints on Railway: one that returns {tier: "free"} for now (expands to pro check later), one that captures an email for the Pro waitlist, one that serves the latest version number for the auto-updater. Total backend is maybe 100 lines. The landing page has one job: explain what Agentis is in two sentences, show a 60-second demo video, and have a download button for Mac and Windows.
